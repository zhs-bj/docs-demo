## 验证
### 感知层
#### 方案分析
1. 方案里明确把本地视觉感知放在很核心的位置，尤其是跌倒检测 / 姿态识别 / 表情情绪识别 / OCR / 多房间监护。
2. 硬件路线也写了 BM1684X / CV186 作为家庭算力中心和感知节点，感知层重点是**本地做姿态识别与表情分析**，原始视频不出网。

#### 盒子里的sophon-demo
1. YOLOv5 / YOLOv10 / YOLOv11_det / SSD / YOLOX：目标检测
2. YOLOv8_pose / OpenPose / HRNet_pose：人体关键点/姿态
3. PP-OCR：文字识别
4. SCRFD / RetinaFace：人脸检测
5. C3D / SlowFast：视频动作识别
6. ByteTrack / DeepSORT：多目标跟踪

#### 各个功能点
1. 用 YOLOv5/YOLOv10 先做“有人/无人”“房间活动检测”
2. 用 YOLOv8_pose/OpenPose 去验证“跌倒检测”的前置能力
3. 用 PP-OCR 去验证“药盒标签、说明书、门牌、快递单”这种识别场景
4. 用 ByteTrack/DeepSORT 去验证“老人从客厅走到厨房”的持续跟踪基础

贴合方案里“感知节点提取特征，中枢节点做理解”的思路

### 边端推理性能
#### 报告里的边端模型推荐
1. Qwen3-8B 作为首选大脑
2. 轻量方案可用 Qwen3-1.5B / Llama 3.2 3B
3. 视觉侧建议 YOLOv10 / YOLOv11-Pose
4. 语音侧建议 Whisper / ChatTTS
4. 部署路线强调 .bmodel、INT4/INT8、多任务并行

#### 如何测定
1. 同一个 demo 多跑几次，记录稳定的 decode / preprocess / inference / postprocess
2. 比较 opencv 版和 bmcv 版
3. 比较 fp32 / int8 的差异
4. 测图片输入和视频输入的差异

这能直接产出后面报告里需要的“算力峰值、内存占用、模型加载耗时”基础数据。

### 多模态接口雏形
现阶段旧盒子里不一定适合直接跑完整大模型栈，但可以做最小闭环原型：
1. 视觉 demo 检测到“人跌倒 / 长时间静止 / 手里拿药盒”
2. OCR demo 识别药盒或标签
3. 再由一个最简单的本地规则引擎输出一句提醒
4. 把结果保存成 json 或文本摘要，模拟“Agent 记忆/家庭摘要”
