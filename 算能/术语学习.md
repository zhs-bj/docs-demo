## airbox盒子
### 尺寸
104mm x 84mm x 50.2mm
1. 说明这台机器体积比较小，属于
- 小型边缘盒子
- 可桌面放置
- 适合嵌入式部署
2. 不是机房服务器，适合用于：
- 工业现场边缘节点
- 本地AI推理盒子
- 小型私有部署设备

### 处理器
SOPHON SG2300X SoC，八核 Arm® Cortex®-A53 (ARMv8) @ 2.3GHz
#### SOPHON SG2300X SoC
1. SoC
- System on Chip，片上系统
- 把 CPU、AI 加速器、接口控制器、存储控制器、多媒体模块等很多功能，集成在一颗芯片里
2. SG2300X
- 算能 的一款 SoC 型号
- 这台 AirBox 的主芯片

#### 八核 Arm Cortex-A53
这颗 SoC 里面有 8 个 ARM CPU 核心，核心类型是 Cortex-A53
1. A53是 ARM 的一种经典低功耗 CPU 核，特点是：
- 功耗低
- 架构成熟
- 通用控制任务够用
- 但不是高性能桌面 CPU
2. 所以这 8 核 CPU 主要不是拿来硬算 AI (NPU/TPU)的，而是拿来负责：负责管事：
- 跑 Linux 系统
- 调度任务
- 管理外设
- 跑控制逻辑
- 调用 AI 加速器
- 做一些前后处理

#### ARMv8 @ 2.3GHz
1. ARMv8：表示 CPU 使用的是 ARM 64 位指令集架构。
2. 3GHz：表示 CPU 主频最高大约 2.3GHz。
主频越高，不一定代表整体 AI 越强，但说明 CPU 本身的通用处理速度还可以。

### 张量处理单元
#### 什么是张量处理单元
1. Tensor Processing Unit或广义上的 NPU / AI Accelerator
2. 专门为神经网络运算设计的硬件单元，张量即多维数组
3. 主要做的事包括：
- 矩阵乘法
- 卷积
- 乘加运算
- 激活相关计算
#### 算力
##### 32TOPS (INT8)
在 8 位整数精度 下，峰值算力可达每秒 32 万亿次运算
1. 适合于**AI推理**：（训练要求的精度高）
- TOPS：整数运算吞吐能力
- INT8：低精度推理模式
2. 适合：边缘视觉任务
- 图像分类
- 目标检测
- OCR，光学字符识别
- 视频分析
##### 16TFLOPS (FP16/BF16)
在 FP16 或 BF16 半精度浮点数 下，峰值算力可达每秒 16 万亿次浮点运算
1. FP16：16 位浮点数，深度学习里很常见。
2. BF16：也是 16 位浮点格式，但指数范围更接近 FP32，很多 AI 框架喜欢用
##### 2TFLOPS (FP32)
在 单精度浮点 FP32 下，峰值算力约为每秒 2 万亿次浮点运算
1. FP32 是比较传统、精度更高的浮点格式。这说明它也能做 FP32 计算，但性能明显低于 INT8/FP16。
2. 精度越高，单次计算越重，同样硬件里能并行塞进去的运算单元越少
3. 说明：这台设备主要定位还是**推理部署**，不是高精度训练平台。

### 支持主流深度学习框架
这台设备的软件工具链，能兼容和接入很多常见 AI 模型来源
1. TensorFlow：Google 的深度学习框架。传统视觉模型很多来自它。
2. Caffe：比较老牌的框架，早期视觉任务里用得多。
3. PyTorch：现在最主流的深度学习框架之一。最常遇到的大概率就是它。
4. Paddle：飞桨，百度的深度学习框架，国内生态里常见。
5. ONNX：
- 不是传统意义上的“训练框架”，更像：模型交换格式 / 中间表示
- 很多模型会走这条链：PyTorch → ONNX → 算能编译工具 → bmodel
6. MXNet：老一代框架，现在少一些。
7. Tengine：偏部署侧轻量推理框架。
8. DarkNet：YOLO 早期版本相关框架。

### 内存：16GB LPDDR4X
#### 内存
1. 内存是：运行时的 RAM
2. 决定：
- 同时能装多大的模型到运行内存
- 能处理多大的输入
- 能开多少路任务
- 多任务并发时会不会爆内存
#### LPDDR4X 
是一种低功耗内存类型：
- 常见于嵌入式设备、手机、边缘设备
- 功耗比普通桌面内存更低
- 适合小型设备
#### 16GB
1. 在边缘盒子里，16GB 不算小，已经是比较实用的容量了。
2. 意味着：
- 跑常规视觉模型基本够用
- 跑一些中等规模多任务推理也有空间
- 对小型 LLM/多模态尝试有一定可能
- 但对特别大的模型仍然有限

### 储存
#### 64GB eMMC
1. 是一种板载闪存，相当于设备内部自带的固态存储，但速度一般比 NVMe SSD 慢。
2. 它主要用来放：操作系统，预装软件，小型模型，配置文件，日志
3. 64GB eMMC 不算特别大，但拿来装系统和一些模型完全够用。
2. 如果要放很多模型、视频数据集、缓存文件，可能就不够宽裕了

#### 16MB SPI 闪存
1. 另一类很小但重要的闪存
2. 通常用来存：Bootloader，启动配置，固件，底层引导程序
3. 不是存视频和模型的，而是给设备开机启动用的关键小存储

#### 提供高速 SD 卡插槽
可以插 SD 卡扩展存储

### 多媒体
非常重要，因为这台机器明显面向**视频 AI 分析**
#### 支持解码32路 H.265/H.264 1080p@25fps 视频
1. 解码：压缩视频流要先**还原成图像帧**才能做 AI 分析，这个过程叫 解码
2. H.265 / H.264：是两种常见视频压缩编码标准
3. 1080p@25fps：分辨率 1920×1080，每秒 25 帧
4. 32路：就是最多能同时处理 32 路这样规格的视频流的解码。
这说明它的视频输入处理能力很强，适合：
- 多摄像头监控
- 多路安防分析
- 工业多相机系统

#### 完全处理32路高清1080P@25fps视频，包括解码和AI分析
1. 能同时完成 解码 + AI推理分析
2. 典型应用是：
- 监控视频智能分析
- 多路摄像头检测
- 边缘安防
- 视频检索
- 行为识别/目标识别

#### 支持编码12路 H.265/H.264 1080p@25fps 视频
编码：把处理后的图像/视频再压缩保存或传输出去

#### JPEG: 1080P@600fps，最大支持32768 x 32768
1. 对 JPEG 图片编解码/处理能力也很强
2. 对于 1080P 级别图片，JPEG 处理吞吐很高
3. 支持非常大的图像尺寸

#### 视频后处理，包括图像 CSC、调整大小、裁剪、填充、边框、字体、对比度和亮度调整
1. CSC：通常指 Color Space Conversion，颜色空间转换。比如：YUV 转 RGB，RGB 转 BGR
2. 很重要，因为现实部署中不只是“丢给模型”，而是通常要经过：
- 解码 → 色彩转换 → resize → 推理 → 画框/标字 → 编码/输出
- 如果这些都能在平台高效完成，整体吞吐会更好。

## 硬件基础
### 存储层次
 从离 CPU 最近、速度最快，到最远、速度最慢，大概是：
寄存器 → 缓存（Cache） → 内存（RAM） → 存储（eMMC / SSD）
#### 寄存器
1. 在 CPU 内部
2. 最快
3. 最小
4. 存当前指令立即要用的数据
#### Cache
1. 也离 CPU 很近
2. 比内存快
3. 比寄存器大
4. 用来减少 CPU 去内存取数据的次数
#### 内存（RAM） 访存说的就是访问内存
1. 系统运行时主工作区
2. 比 Cache 慢，但容量大很多
3. 正在运行的程序和数据主要在这里
#### 存储（eMMC/SSD）
1. 长期保存文件
2. 断电不丢
3. 容量更大
4. 速度通常比 RAM 慢得多

## 关于AI
### 视频解码
#### 图像帧
1. 就是视频中的单张图片
2. 是看得见、能直接拿来做 AI 分析的数据形式
#### 压缩视频流
1. 把很多连续图像帧，按照某种视频编码标准压缩后形成的数据流
2. 比如：H.264，H.265。这些标准不会简单地把每一帧原封不动存起来，因为那样太大了
3. 会利用很多方法压缩，比如：
- 去掉重复信息
- 利用前后帧相似性
- 只记录变化部分
4. 所以压缩后得到的是一种：不适合人直接看像素，也不能直接拿去喂给大多数视觉模型，但很适合传输和存储的数据形式。

### 视频AI分析
1. 让 AI 自动去“看”视频内容，并从中提取有用信息
2. 常见任务
- 目标检测
- 分类
- 跟踪
- 行为识别
- OCR/文字识别
- 计数/统计
